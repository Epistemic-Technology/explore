When navigating to a symbol in the tree view, the source pane now scrolls intelligently: if the symbol body fits in the window, it scrolls minimally to make it visible; if it doesn't fit, the symbol's first line anchors to the top of the view. This scroll-into-view logic was applied to both forward navigation (tree cursor changes) and back/forward stack navigation. The change ensures symbols remain in focus without unnecessary scrolling.
